Tips

Take extra care while using ordred list or unordered list

Identify the menu:

Use role="navigation" or <nav> tag or both

Labeling navigations:

Using a (hidden) heading inside nav tag like <h2>Main menu</h2>

Do not make hidden with display none or visibility hidden. Screen reader don't read hidden content. Hide with font size 0 or other techniques

Labeling navigations:

aria-label i.e aria-label="Main Menu"

<nav role="navigation" aria-label="Main Menu">

aria-labelledby

<nav role="navigation" aria-labelledby="mainmenulabel">

Indicating the current page

<span class="visuallyhidden">Current Page: </span>

If menu has sub menu, then it should indicate the menu visually ie. arrow down icon

Sub menu should close after one second

If menu has submenu use aria-haspopup="true"

Use aria-expanded

Keyboard

Use tab key for iterating top label menu

Use up and down array for iterating sub menu

Web app Menu (Strictly Web app)

role="menubar" : horizontal menu

role="menu": Represents a set of links or commands in a menu bar, it is used for the fly-out menus

role="menuitem": Represents an individual menu item.

Key mapping for top-level menu items

Keys for top-level menu items
Key Action
tab Select the next top-level menu item
right arrow
shift + tab Select the previous top-level menu item
left arrow
return/enter Open the submenu, select first submenu item.
space
down arrow
up arrow Open the submenu, select last submenu item.
esc Leave the menu
Keys for submenu items
Key Action
tab Close the submenu, select the next top-level menu item
right arrow
shift + tab Close the submenu, select the previous top-level menu item
left arrow
return/enter Carry out function of this item. (In this example: bring up a dialog box with the text of the chosen menu item.)
space
down arrow Select next submenu item
up arrow Select previous submenu item
esc Close the submenu, select the current top-level menu item
